30 - HTML Form Attributes

30.1 - O atributo action

O atributo action define a ação a ser executada quando o formulário for enviado.

Normalmente, os dados do formulário são enviados para um arquivo no servidor quando o usuário clica no botão enviar.

Se o atributo action for omitido, a ação será definida para a página atual.

No exemplo abaixo, os dados do formulário são enviados para um arquivo chamado “action_page.php”. Este arquivo contém um script do lado do servidor que trata os dados do formulário.

Código:

<form action="/action_page.php">
    <label for="nome">Nome:</label><br>
    <input type="text" id="nome" name="nome" value="John"><br>
    <label for="sobrenome">Sobrenome:</label><br>
    <input type="text" id="sobrenome" name="sobrenome" value="Doe"><br><br>
    <input type="submit" value="Enviar">
</form>

30.2 - O atributo target

O atributo target especifica onde exibir a resposta recebida após o envio do formulário.

O targetatributo pode ter um dos seguintes valores:

Valor Descrição
_em branco A resposta é exibida em uma nova janela ou guia
_self A resposta é exibida na janela atual
_parent A resposta é exibida no quadro pai
_top A resposta é exibida em todo o corpo da janela
nome do quadro A resposta é exibida em um iframe nomeado

Código:

<form action="/action_page.php" target="_blank">
    <label for="nome">Nome:</label><br>
    <input type="text" id="nome" name="nome" value="John"><br>
    <label for="sobrenome">Sobrenome:</label><br>
    <input type="text" id="sobrenome" name="sobrenome" value="Doe"><br><br>
    <input type="submit" value="Enviar">
</form>

30.3 - O atributo method

O atributo method especifica o método HTTP a ser usado ao enviar os dados do formulário.

Os dados do formulário podem ser enviados como variáveis ​​​​de URL (com method="get") ou como pós-transação HTTP (com method="post").

O método HTTP padrão ao enviar dados de formulário é GET. Porém sempre use POST se os dados do formulário contiverem informações confidenciais ou pessoais.

Enviar via GET:

<form action="/action_page.php" method="get">
    <label for="nome">Nome:</label><br>
    <input type="text" id="nome" name="nome" value="John"><br>
    <label for="sobrenome">Sobrenome:</label><br>
    <input type="text" id="sobrenome" name="sobrenome" value="Doe"><br><br>
    <input type="submit" value="Enviar">
</form>

Exemplo:






Informações sobre o GET

  • Acrescenta os dados do formulário ao URL, em pares nome/valor
  • NUNCA use GET para enviar dados confidenciais! (os dados do formulário enviado ficam visíveis no URL!)
  • O comprimento de um URL é limitado (2.048 caracteres)
  • Útil para envios de formulários onde um usuário deseja marcar o resultado
  • GET é bom para dados não seguros, como strings de consulta no Google

Enviar via POST:

<form action="/action_page.php" method="post">
    <label for="nome">Nome:</label><br>
    <input type="text" id="nome" name="nome" value="John"><br>
    <label for="sobrenome">Sobrenome:</label><br>
    <input type="text" id="sobrenome" name="sobrenome" value="Doe"><br><br>
    <input type="submit" value="Enviar">
</form>

Exemplo:






Informações sobre o POST

  • Acrescenta os dados do formulário ao corpo da solicitação HTTP (os dados do formulário enviado não são mostrados na URL)
  • POST não tem limitações de tamanho e pode ser usado para enviar grandes quantidades de dados.
  • Os envios de formulários com POST não podem ser marcados como favoritos

30.4 - O atributo autocomplete

O atributo autocomplete especifica se um formulário deve ter o preenchimento automático ativado ou desativado.

Quando o preenchimento automático está ativado, o navegador completa automaticamente os valores com base nos valores inseridos anteriormente pelo usuário.

Código:

<form action="/action_page.php" autocomplete="on">
    <label for="nome">Nome:</label><br>
    <input type="text" id="nome" name="nome"><br>
    <label for="sobrenome">Sobrenome:</label><br>
    <input type="text" id="sobrenome" name="sobrenome"><br><br>
    <input type="submit" value="Enviar">
</form>

30.5 - O atributo novalidate

O novalidate é um atributo booleano. Quando presente, especifica que os dados do formulário (entrada) não devem ser validados quando submetidos.

Código:

<form action="/action_page.php" novalidate>
    <label for="nome">Nome:</label><br>
    <input type="text" id="nome" name="nome" value="John"><br>
    <label for="sobrenome">Sobrenome:</label><br>
    <input type="text" id="sobrenome" name="sobrenome" value="Doe"><br><br>
    <input type="submit" value="Enviar">
</form>
