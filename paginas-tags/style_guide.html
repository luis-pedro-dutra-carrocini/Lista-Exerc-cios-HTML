22 - HTML Style Guide

22.1 - Introdução

Um código HTML consistente, limpo e organizado torna mais fácil para outras pessoas lerem e compreenderem seu código.

Aqui estão algumas diretrizes e dicas para criar um bom código HTML.


22.2 - Sempre declarar o tipo de documento

Sempre declare o tipo de documento como a primeira linha do seu documento.

Exemplo:

<!DOCTYPE html>

22.3 - Use nomes de elementos em letras minúsculas

HTML permite misturar letras maiúsculas e minúsculas nos nomes dos elementos. No entanto, recomendamos usar nomes de elementos em letras minúsculas porque:

  • Misturar nomes maiúsculos e minúsculos parece ruim
  • Os desenvolvedores normalmente usam nomes em minúsculas
  • Letras minúsculas parecem mais limpas
  • Letras minúsculas são mais fáceis de escrever

Bom:

<p>Isso é um parágrafo</p>

Ruim:

<P>Isso é um parágrafo</P>

22.4 - Fechar todos os elementos HTML

Em HTML, você não precisa fechar todos os elementos (por exemplo, o elemento <p>). No entanto, recomendamos fortemente fechar todos os elementos HTML, assim:

Bom:

<section> <p>Isso é um parágrafo</p> <p>Isso é um parágrafo</p> </section>

Ruim:

<section> <p>Isso é um parágrafo <p>Isso é um parágrafo </section>

22.5 - Use nomes de atributos em letras minúsculas

HTML permite misturar letras maiúsculas e minúsculas em nomes de atributos. No entanto, recomendamos usar nomes de atributos em letras minúsculas porque:

  • Misturar nomes maiúsculos e minúsculos parece ruim
  • Os desenvolvedores normalmente usam nomes em minúsculas
  • Letras minúsculas parecem mais limpas
  • Letras minúsculas são mais fáceis de escrever

Bom:

<a href="https://www.w3schools.com/html/">Visite o site W3 Schools</a>

Ruim:

<a HREF="https://www.w3schools.com/html/">Visite o site W3 Schools</a>

22.6 - Sempre citar valores de atributos

HTML permite valores de atributos sem aspas. No entanto, recomendamos citar valores de atributos porque:

  • Os desenvolvedores normalmente citam valores de atributos
  • Os valores citados são mais fáceis de ler
  • Letras minúsculas parecem mais limpas
  • Você DEVE usar aspas se o valor contiver espaços

Bom:

<table class="striped">

Ruim:

<table class=striped>

Muito Ruim:

<table class=table striped>

22.7 - Sempre especifique alt, largura e altura para imagens

Sempre especifique o altatributo para imagens. Este atributo é importante se a imagem, por algum motivo, não puder ser exibida.

Além disso, sempre defina o widthe heightdas imagens. Isso reduz a oscilação, pois o navegador pode reservar espaço para a imagem antes de carregá-la.

Bom:

<img src="html5.gif" alt="HTML5" style="width:128px;height:128px">

Ruim:

<img src="html5.gif">

22.8 - Espaços e sinais de igual

HTML permite espaços em torno de sinais de igual. Mas sem espaço é mais fácil de ler e agrupa melhor as entidades.

Bom:

<link rel="stylesheet" href="styles.css">

Ruim:

<link rel = "stylesheet" href = "styles.css">

22.9 - Evite longas linhas de código

Ao usar um editor HTML, NÃO é conveniente rolar para a direita e para a esquerda para ler o código HTML. Tente evitar linhas de código muito longas.


22.10 - Linhas em branco e recuo

Não adicione linhas em branco, espaços ou recuos sem motivo. Para facilitar a leitura, adicione linhas em branco para separar blocos de código grandes ou lógicos. Para facilitar a leitura, adicione dois espaços de recuo ouu tab.

Bom:

<h1>Famous Cities</h1>

<h2>Tokyo</h2>
<p>Tokyo is the capital of Japan, the center of the Greater Tokyo Area, and the most populous metropolitan area in the world.</p>

<h2>London</h2>
<p>London is the capital city of England. It is the most populous city in the United Kingdom.</p>

<h2>Paris</h2>
<p>Paris is the capital of France. The Paris area is one of the largest population centers in Europe.</p>

Ruim:

<h1>Famous Cities</h1>
<h2>Tokyo</h2><p>Tokyo is the capital of Japan, the center of the Greater Tokyo Area, and the most populous metropolitan area in the world.</p>
<h2>London</h2><p>London is the capital city of England. It is the most populous city in the United Kingdom.</p>
<h2>Paris</h2><p>Paris is the capital of France. The Paris area is one of the largest population centers in Europe.</p>

Bom exemplo de tabela:

<table>
  <tr>
    <th>Name</th>
    <th>Description</th>
  </tr>
  <tr>
    <td>A</td>
    <td>Description of A</td>
  </tr>
  <tr>
    <td>B</td>
    <td>Description of B</td>
  </tr>
</table>

Bom exemplo de lista:

<ul>
  <li>London</li>
  <li>Paris</li>
  <li>Tokyo</li>
</ul>

22.11 - Nunca pule o elemento <title>

O conteúdo do título de uma página é muito importante para a otimização de mecanismos de pesquisa (SEO)! O título da página é usado pelos algoritmos do mecanismo de pesquisa para decidir a ordem ao listar as páginas nos resultados da pesquisa. Tente tornar o título o mais preciso e significativo possível.

Sempre:

<title>HTML Style Guide and Coding Convention</title>

22.12 - Nunca omita <html> e <body>

Uma página HTML será validada sem as tags <html> e <body>. No entanto, é altamente recomendável sempre adiciona-las. A omissão <body> pode produzir erros em navegadores mais antigos. Omitir <html>e <body> também pode travar o software DOM e XML.


22.13 - Nunca omita <head>

A tag HTML <head> também pode ser omitida. Os navegadores adicionarão todos os elementos antes de <body>, a um <head> elemento padrão. No entanto, recomendamos usa-lo.


22.14 - Deve-se fechar elementos HTML vazios?

Em HTML, é opcional fechar elementos vazios. Se você espera que o software XML/XHTML acesse sua página, mantenha a barra de fechamento (/), pois ela é obrigatória em XML e XHTML.

Permitido:

<meta charset="utf-8">

Permitido:

<meta charset="utf-8" />

22.15 - Adicione o atributo lang

Você deve sempre incluir o langatributo dentro da <html> tag, para declarar o idioma da página web. O objetivo é ajudar os mecanismos de pesquisa e navegadores.

Exemplo:

<!DOCTYPE html>
<html lang="en-us">
<head>
  <title>Page Title</title>
</head>
<body>

  <h1>This is a heading</h1>
  <p>This is a paragraph.</p>

</body>
</html>

22.16 - Metadados

Para garantir a interpretação adequada e a indexação correta do mecanismo de pesquisa, tanto o idioma quanto a codificação de caracteres devem ser definidos o mais cedo possível em um documento HTML: <meta charset="charset">

Exemplo:

<!DOCTYPE html>
<html lang="en-us">
<head>
  <meta charset="UTF-8">
  <title>Título</title>
</head>

22.17 - Configurando a janela de visualização

A janela de visualização é a área visível do usuário em uma página da web. Varia de acordo com o dispositivo, será menor em um telefone celular do que na tela de um computador.

Você deve incluir o seguinte elemento <meta> em todas as suas páginas da web:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Isso fornece instruções ao navegador sobre como controlar as dimensões e o dimensionamento da página.

A width=device-widthparte define a largura da página para seguir a largura da tela do dispositivo (que varia dependendo do dispositivo).

A initial-scale=1.0parte define o nível de zoom inicial quando a página é carregada pela primeira vez pelo navegador.


22.18 - Comentários

Comentários curtos devem ser escritos em uma linha, assim:

<!-- Um comentário -->

Comentários longos são mais fáceis de observar se forem recuados com dois espaços, devem ser escritos assim:

<!--
  This is a long comment example. This is a long comment example.
  This is a long comment example. This is a long comment example.
-->

22.19 - Usando folhas de estilo

Use sintaxe simples para vincular folhas de estilo (o typeatributo não é necessário):

<link rel="stylesheet" href="styles.css">

Regras CSS curtas podem ser escritas compactadas, assim:

p.intro {font-family:Verdana;font-size:16em;}

Regras CSS curtas podem ser escritas compactadas, assim:

body {
  background-color: lightgrey;
  font-family: "Arial Black", Helvetica, sans-serif;
  font-size: 16em;
  color: black;
}

22.20 - Carregando JavaScript em HTML

Use sintaxe simples para carregar scripts externos (o typeatributo não é necessário):

<script src="myscript.js">

22.21 - Acessando elementos HTML com JavaScript

Usar código HTML "desarrumado" pode resultar em erros de JavaScript. Estas duas instruções JavaScript produzirão resultados diferentes:

getElementById("Demo").innerHTML = "Hello";

getElementById("demo").innerHTML = "Hello";

22.22 - Use nomes de arquivos em letras minúsculas

Alguns servidores web (Apache, Unix) diferenciam maiúsculas de minúsculas em relação aos nomes de arquivos: "london.jpg" não pode ser acessado como "London.jpg".

Outros servidores web (Microsoft, IIS) não diferenciam maiúsculas de minúsculas: "london.jpg" pode ser acessado como "London.jpg".

Se você usar uma mistura de letras maiúsculas e minúsculas, deve estar ciente disso.

Se você passar de um servidor que não diferencia maiúsculas de minúsculas para um que diferencia maiúsculas de minúsculas, mesmo pequenos erros danificarão sua web!

Para evitar esses problemas, use sempre nomes de arquivos em letras minúsculas!


22.23 - Extensões de arquivo

Os arquivos HTML devem ter uma extensão .html (.htm é permitido)

Os arquivos CSS devem ter uma extensão .css

Os arquivos JavaScript devem ter uma extensão .js


22.24 - Diferenças entre .htm e .html

Não há diferença entre as extensões de arquivo .htm e .html! Ambos serão tratados como HTML por qualquer navegador e servidor web.


22.25 - Nomes de arquivos padrão

Quando uma URL não especifica um nome de arquivo no final (como "https://www.w3schools.com/"), o servidor apenas adiciona um nome de arquivo padrão, como "index.html", "index.htm", " default.html" ou "default.htm".

Se o seu servidor estiver configurado apenas com "index.html" como nome de arquivo padrão, seu arquivo deverá ser denominado "index.html" e não "default.html".

Entretanto, os servidores podem ser configurados com mais de um nome de arquivo padrão; normalmente você pode configurar quantos nomes de arquivos padrão desejar.
