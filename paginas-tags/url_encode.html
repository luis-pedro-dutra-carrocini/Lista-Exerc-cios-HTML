27.1 - Localizadores uniformes de recursos HTML

URL é outra palavra para endereço da web. Uma URL pode ser composta por palavras (por exemplo, w3schools.com) ou por um endereço de protocolo de Internet (IP) (por exemplo, 192.68.20.50). A maioria das pessoas insere o nome quando navega, porque os nomes são mais fáceis de lembrar do que os números

27.2 - URL - Localizador Uniforme de Recursos

Os navegadores da Web solicitam páginas de servidores da Web usando um URL. Um Uniform Resource Locator (URL) é usado para endereçar um documento (ou outros dados) na web.

Um endereço da web como https://www.w3schools.com/html/default.asp segue estas regras de sintaxe: esquema://prefixo.dominio:porta/caminho/nomedoarquivo

Explicação:

  • esquema - define o tipo de serviço de Internet (o mais comum é http ou https)
  • prefixo - define um prefixo de domínio (o padrão para http é www )
  • domínio - define o nome de domínio da Internet (como w3schools.com)
  • porta - define o número da porta no host (o padrão para http é 80)
  • path - define um caminho no servidor (se omitido: o diretório raiz do site)
  • nome do arquivo - define o nome de um documento ou recurso

27.3 - Codificação de URL

URLs só podem ser enviados pela Internet usando o conjunto de caracteres ASCII . Se um URL contiver caracteres fora do conjunto ASCII, o URL deverá ser convertido.

A codificação de URL converte caracteres não ASCII em um formato que pode ser transmitido pela Internet. Substituindo caracteres não ASCII por "%" seguido de dígitos hexadecimais. URLs também não podem conter espaços, então a codificação de URL normalmente substitui um espaço por um sinal de mais (+) ou% 20.

26.4 - O conjunto de caracteres ISO-8859-1

ISO-8859-1 era o conjunto de caracteres padrão para HTML 4. Este conjunto de caracteres suportava 256 códigos de caracteres diferentes. HTML 4 também suporta UTF-8.

  • Idêntico ao ASCII para os primeiros 127 caracteres
  • Não usa os caracteres de 128 a 159
  • Idêntico a ANSI e UTF-8 de 160 a 255

HTML4:

<meta http-equiv="Content-Type" content="text/html;charset=ISO-8859-1">

HTML5:

<meta charset="ISO-8859-1">

26.5 - O conjunto de caracteres UTF-8

  • É idêntico ao ASCII para os valores de 0 a 127
  • Não usa os caracteres de 128 a 159
  • Idêntico ao ANSI e 8859-1 de 160 a 255
  • Contínua do valor 256 até 10.000 caracteres

<meta charset="UTF-8">
