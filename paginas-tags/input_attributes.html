33 - HTML Input Attributes

33.1 - Atributo Value

O atributo value especifica um valor inicial para um campo de entrada:

Código:

<form>
   <label for="fname">Nome:</label><br>
   <input type="text" id="fname" name="fname" value="John"><br>
   <label for="lname">Sobrenome:</label><br>
   <input type="text" id="lname" name="lname" value="Doe">
</form>

33.2 - Atributo Readonly

O atributo readonly especifica que um campo de entrada é somente leitura. Um campo de entrada somente leitura não pode ser modificado (no entanto, um usuário pode tabulá-lo, destacá-lo e copiar o texto dele). O valor de um campo de entrada somente leitura será enviado ao enviar o formulário!

Código:

<form>
   <label for="fname">Nome:</label><br>
   <input type="text" id="fname" name="fname" value="John" readonly><br>
   <label for="lname">Sobrenome:</label><br>
   <input type="text" id="lname" name="lname" value="Doe">
</form>

33.3 - Atributo Disabled

O atributo input disabled especifica que um campo de entrada deve ser desabilitado. Um campo de entrada desabilitado é inutilizável e não clicável. O valor de um campo de entrada desabilitado não será enviado ao enviar o formulário!

Código:

<form>
   <label for="fname">Nome:</label><br>
   <input type="text" id="fname" name="fname" value="John" disabled><br>
   <label for="lname">Sobrenome:</label><br>
   <input type="text" id="lname" name="lname" value="Doe">
</form>

33.4 - Atriibuto Size

O atributo

size

especifica a largura visível, em caracteres, de um campo de entrada.>

O valor padrão para sizeé 20. Esse atributo funciona com os seguintes tipos de entrada: texto, pesquisa, tel, url, email e senha.

33.5 - Atributo maxlength

O atributo

maxlength

especifica o número máximo de caracteres permitidos em um campo de entrada.

Quando o maxlength é definido, o campo de entrada não aceitará mais do que o número especificado de caracteres. No entanto, este atributo não fornece nenhum feedback. Portanto, se quiser alertar o usuário, você deve escrever o código JavaScript.

Código:

<form>
   <label for="fname">Nome:</label><br>
   <input type="text" id="fname" name="fname" size="50"><br>
   <label for="pin">PIN:</label><br>
   <input type="text" id="pin" name="pin" maxlength="4" size="4">
</form>

33.6 - Atributos min e max

Os atributos min e max, especificam os valores mínimo e máximo para um campo de entrada. Eles funcionam com os seguintes tipos de entrada: número, intervalo, data, data e hora local, mês, hora e semana. Use os atributos max e min juntos para criar um intervalo de valores legais.

Código:

<form>
   <label for="datemax">Data antes de 1980:</label>
   <input type="date" id="datemax" name="datemax" max="1979-12-31"><br><br>

   <label for="datemin">Data depois de 2000:</label>
   <input type="date" id="datemin" name="datemin" min="2000-01-02"><br><br>

   <label for="quantity">Quantidade (de 1 a 5):</label>
   <input type="number" id="quantity" name="quantity" min="1" max="5">
</form>

33.7 - Atributo multiple

O atributo multiple especifica que o usuário pode inserir mais de um valor em um campo de entrada. Ele funciona com os seguintes tipos de entrada: email e arquivo.

Código:

<form>
   <label for="files">Selecione os arquivos:</label>
   <input type="file" id="files" name="files" multiple>
</form>

Exemplo:


33.8 - Atributo pattern

O atributo pattern especifica uma expressão regular com a qual o valor do campo de entrada é verificado quando o formulário é enviado. O pattern funciona com os seguintes tipos de entrada: texto, data, pesquisa, url, tel, email e senha.

Use o atributo global title para descrever o padrão para ajudar o usuário.

Código:

<form>
   <label for="country_code">Código do país:</label>
   <input type="text" id="country_code" name="country_code">
   pattern="[A-Za-z]{3}" title="Código do país de três letras">
</form>

33.10 - Atributo placeholder

O atributo placeholder especifica uma dica curta que descreve o valor esperado de um campo de entrada (um valor de amostra ou uma breve descrição do formato esperado). A dica curta é exibida no campo de entrada antes que o usuário insira um valor.

O atributo funciona com os seguintes tipos de entrada: texto, pesquisa, url, número, tel, email e senha.

Código:

<form>
   <label for="phone">Insira o número de telefone:</label>
   <input type="tel" id="phone" name="phone"
   placeholder="(16) 99272-3654"
   pattern="[0-9]{3}-[0-9]{2}-[0-9]{3}">
</form>

33.11 - Atributo required

O atributo required especifica que um campo de entrada deve ser preenchido antes de enviar o formulário.

O atributo funciona com os seguintes tipos de entrada: texto, pesquisa, url, tel, email, senha, seletores de data, número, caixa de seleção, rádio e arquivo.

Código:

<form>
   <label for="username">Nome:</label>
   <input type="text" id="username" name="username" required>
</form>

33.12 - Atributo step

O atributo step especifica os intervalos numéricos legais para um campo de entrada.

Exemplo: se step="3", os números legais podem ser -3, 0, 3, 6, etc.

O atributo funciona com os seguintes tipos de entrada: número, intervalo, data, data e hora local, mês, hora e semana.

As restrições de entrada não são infalíveis e o JavaScript oferece muitas maneiras de adicionar entradas ilegais. Para restringir a entrada com segurança, ela também deve ser verificada pelo receptor (o servidor)!

Código:

<form>
   <label for="points">Pontos:</label>
   <input type="number" id="points" name="points" step="3">
</form>

33.14 - Atributs height e width

Os atributos height e width especificam a altura e a largura de um <input type="image">elemento.

Sempre especifique os atributos de altura e largura das imagens. Se a altura e a largura forem definidas, o espaço necessário para a imagem será reservado quando a página for carregada. Sem esses atributos, o navegador não sabe o tamanho da imagem e não pode reservar o espaço apropriado para ela. O efeito será que o layout da página mudará durante o carregamento (enquanto as imagens são carregadas).

Código:

<form>
   <label for="fname">Nome:</label>
   <input type="text" id="fname" name="fname"><br><br>
   <label for="lname">Sobrenome:</label>
   <input type="text" id="lname" name="lname"><br><br>
   <input type="image" src="img/img_submit.gif" alt="Ennviar" width="48" height="48">
</form>

33.15 - Atributo list

O atributo list refere-se a um <datalist> elemento que contém opções predefinidas para um elemento <input>.

Código:

      <option value="Firefox">
      <option value="Chrome">

   </datalist>
</form>

33.16 - Atributo autocomplete

O atributo autocomplete especifica se um formulário ou campo de entrada deve ter o preenchimento automático ativado ou desativado.

O preenchimento automático permite que o navegador preveja o valor. Quando um usuário começa a digitar em um campo, o navegador deve exibir opções para preencher o campo, com base nos valores digitados anteriormente.

O atributo funciona com <form> os seguintes <input> tipos: texto, pesquisa, url, tel, email, senha, selecionadores de data, intervalo e cor.

Em alguns navegadores pode ser necessário ativar uma função de preenchimento automático para que isso funcione (procure em “Preferências” no menu do navegador).

Código:

<form action="/action_page.php" autocomplete="on">
   <label for="fname">Nomme:</label>
   <input type="text" id="fname" name="fname"><br><br>
   <label for="lname">Sobrenome:</label>
   <input type="text" id="lname" name="lname"><br><br>
   <label for="email">Email:</label>
   <input type="email" id="email" name="email" autocomplete="off"><br><br>
   <input type="submit" value="Submit">
</form>
